# ADR-002: SVGアイコンを活用したミニゲームの導入

## ステータス
提案中

## コンテキスト
現在のプロジェクトには多数のAWS SVGアイコンが存在しており、これらを活用したユーザー体験の向上が求められています。クイズ機能の導入前に、SVGの軽量さと鮮明さを活かしたインタラクティブなミニゲーム（アイコンのキャッチやマッチングなど）を導入することで、ユーザーが楽しみながらアイコンに親しむ機会を提供します。

## 決定事項
1. **ゲーム内容**: 
   - 画面上部からランダムにAWSアイコン（SVG）が落下し、それをキャッチまたはクリックするシンプルなアクションゲーム。
   - 初期段階では、データセットが豊富なAWSアイコンのみを使用する。
2. **技術選定**:
   - 描画: HTML5 Canvas または DOM (SVG要素の直接操作)。軽量性を重視し、外部ライブラリは使用せず Vanilla JS で実装する。
   - アセット: プロジェクト内の `img/` ディレクトリにある既存の SVG アイコンを直接利用する。
3. **配置**:
   - `game/` ディレクトリを新設し、`index.html`, `game.js`, `style.css` を配置する。
   - Vercel の `vercel.json` に `/game` へのルーティングを追加する。

## 議論
- **なぜAWSのみか？**: AWSアイコンのディレクトリ構造が整理されており、SVGファイルが豊富に揃っているため、プロトタイプ開発に適している。
- **パフォーマンス**: SVGを大量に動かす際のパフォーマンス懸念があるが、現代のブラウザでは数百個程度であればDOM操作でも十分高速である。必要に応じて Canvas 描画へ移行する。

## コンプライアンス
- アイコンの利用ガイドライン（AWS等）を遵守し、商用利用やロゴの改変に当たらない範囲でゲーム化する。

## 成果
- ユーザーがクイズを始める前の「ウォーミングアップ」として機能する。
- SVGアセットの有効活用。
